## Phase 1 — RAW

v4.0.2 is shipped (5 OS installers on public, auto-updater works,
132/132 audits green at ship time).  User launched the dev build
and immediately found four issues:

1. **Claude (Chat) profile exits with code 1.**  Terminal shows
   `prompt argument when using --print` from the CLI, then
   `[process exited with code 1]`.  The yellow stream-json
   diagnostic I added in v4.0.2 (`EmbeddedTerminal.tsx` fast-exit
   handler that detects `profile === 'api.anthropic.claude-chat'`
   + non-zero exit within 6s) is NOT visible.  So either the
   listener doesn't fire fast enough OR the profile check is
   incorrect OR my edit got reverted somewhere.

2. **Curated research model "✓ Launched" state shown** for
   `bartowski/Llama-3.2-3B-Instruct-uncensored-GGUF` Q4_K_M.
   The card state is correct — but the user flagged it as a
   problem.  Almost certainly the issue is downstream of the
   click: Ollama didn't pull (probably not running), OR the
   spawned Models tab exited fast with code 1.

3. **JavaScript error popup in main process**:
   `Cannot resize a pty that has already exited`
   Stack: `WindowsPtyAgent.resize` → `WindowsTerminal.resize` →
   `PtyManager.resize` → `PtyRegistry.resize` → `IpcMainImpl`.
   This is a real crash that surfaces to the user as a modal
   error dialog.  Caused by renderer firing resize events to a
   PTY that already exited.

4. **Commands panel for Claude (Chat) is just an empty-state
   message** with no actionable affordance.

## Phase 2 — NODES

- The PTY resize crash is most urgent — it surfaces a modal at
  the user even when they're not actively engaging with that
  terminal.
- The Claude (Chat) bug has been "fixed" in v4.0.1 (cli-resolver)
  and v4.0.2 (diagnostic) but the underlying spawn behaviour
  still rejects.  Root cause is the local Claude CLI's actual
  handling of `--input-format=stream-json`.
- Audit gap: my CDP harnesses didn't exercise terminal
  lifecycle (spawn → exit → resize) so they missed the PTY
  crash class entirely.
- Commands panel polish is real but lowest priority.

## Phase 3 — REFLECT

The shipped v4.0.2 is honest about what was tested (UI panels +
IPC calls).  PTY lifecycle is a code path scripted audits won't
hit without a fundamentally different harness.  Lesson for
v4.0.3: add a long-running-terminal smoke that spawns Claude,
lets it run 30s, kills it, fires resize after death, confirms
no crash.
